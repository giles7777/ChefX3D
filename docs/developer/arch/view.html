View Design

A view is an editable display of the world model. Common types of views might be 2D Image, 2D Vector, or 3D view. Multiple views might be active in a viewing application at once.

Concepts

View Properties

In addition to the properties an entity has defined, there will also be a set properties defined for views. These properties are editable while in the view. They include the entities location and orientation. Its acceptable for a property to exist in both places.

Selection Set

All views share a common selection set. Some tools will operate on this set.

Undo/Redo

A view shall be able to undo/redo changes. The command list will be a global list of changes made to the model. Anything not changing the model will not be affected, ie things like turning on wireframe mode.

Placement Help

A view might implement a snap-to functionality. Or a feature which helps place objects in a tight space. A space planner application might use this to help pack objects into a 3D space without any of the objects penetrating or wasting space.

Implementation

Views will be the largest single piece of code in the system. The library user can implement their own views, but its expected that most users will use the premade versions.

Model Updates

All changes made in a view must be communicated to the model.
