System Architecture

This toolkit is designed to make writing content specific editing tools easy. It should be UI toolkit agnostic and likely will contain both Swing and SWT implementations.

Major concepts in the design:

Concept Definitions:

  • Tool - An authoring tool
    • Tool Descriptor - Outside description of a tool
    • Tool Group - A grouping of tools
  • Entity - An instance of an authoring tool and a set of properties
  • Classification level - The classification of a tool(security, license, cost)
    • Just need an integer for each string
    • Have an icon for each displayable on the toolbar
Components:
  • World Model - The collection of current entities
  • Property Editor - an editor of entity properties
  • Toolbar - A collection of tools and a method to select them
  • View - A view onto the model. Authoring operations can be performed on this view.
  • Importer - Populates the model with entities from some source
  • Exporter - Writes out the entities to some format

Customization

Ways the user can customize:
  • What tools are provided and their naming/icons
  • Layout of the editor components
  • Implementation of Toolbar, View, Property Editor and Model
  • Project specific conversion of model data into file formats

User Interface

The library should provide a reasonable implementation for all required pieces. The average project should not need to replace these. Some control over look and feel should be provided to make it possible for most projects to use these implementations.

Default implementations for:

  • View(2D, 3D)
  • Toolbar
  • Property Editor
