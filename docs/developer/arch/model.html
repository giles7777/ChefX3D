World Model Design

The model is the backing store for the current list of entities. It receives notification of all changes to entities.

Concepts

The simplist Model might store the the list in memory locally. More complex models might synchronize distributed updates to a shared model.

All changes to the model should be absolute changes. No delta changes like positionAdded(). This allows the network distribution of the changes.

Undo/Redo

A model shall be able to undo/redo changes. The command list will be a global list of changes made to the model.
