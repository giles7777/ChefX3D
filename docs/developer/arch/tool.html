Tool Design

A tool is an authoring widget which can create entities. How that is accomplisted is dependent on the tool.

Concepts

Tool Types

The tool panel is where tools are selected for authoring. Tool operation is determined by its tool type. The following tool types are supported:

  • World - Specifies the base location
  • Model - A locatable visual model
  • MultiSegment - A set of lines
  • Building - Building design tool
  • Helper - A helper tool
  • Controller - Controlls other entities

World

A scenario has one base location of operations. The model is expected to be located in world coordinates and cannot be transformed. A location should provide the input needed for all application views. If its not provided then the application will not disable that view.

A world may have a parameter space.

Model

A Model is a tool which can be positioned in 3 space. Its created by dragging the model onto the Main Window. This will create a new Entity using that model as the visual. A model is backed by an X3D model and contains an XML description of its parameter space. Those parameters are modified in the Property editor. A behavior can be associated with an entity to drive it.

MultiSegment

A multi-segment tool is composed of several segments. The combined object can have a set of properties. Another set of properties(Segement Properties) are available on each segment. Authoring continues to create new segments until the tool is exited.

Selection

A Selection tool performs an operation on the current selection set. After the operation is performed the selection set will be cleared.

Building

The building tool is for authoring rough building models. This will handle authoring a floor plan, the number of stories and a simple roof line. A simple texturing panel will allow textures for the building sides and roof. TODO: Not sure this is a really a tool type.

Helper

A helper tool is not a standalone entity. It is edited exactly like a Model, but it will not be treated the same for export.

Controller

An entity which controls other entities. The location of this entity does not matter. An example might be a Time device or the end conditions for a simulation.

Implementation

Every View in the system will need to understand how to render each tool type. Each tool will have at least a 2D image representation. A view that has no specific understanding of a tool can use this to render it.
